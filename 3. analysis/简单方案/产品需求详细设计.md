## 二、功能详细设计

### 2.1 项目信息管理

#### 2.1.1 功能概述
**目标**：实现项目全生命周期的基础信息管理，支持项目的创建、编辑、查看、状态跟踪等操作。

#### 2.1.2 核心功能

**项目基础信息管理**
- **项目名称**：文本输入，必填项，最大长度100字符
- **承接日期**：日期选择器，必填项
- **项目概况**：多行文本输入，选填项，最大长度500字符
- **出图单位**：文本输入，必填项，最大长度50字符
- **项目编号**：文本输入，自动生成或手动输入，唯一性校验

**项目状态管理**
- **状态类型**：立项、进行中、完成、归档
- **状态变更**：支持状态流转，记录变更时间和操作人
- **进度跟踪**：显示项目当前阶段和完成情况

**项目变更管理**
- **变更记录**：记录变更内容、变更日期、变更原因
- **变更历史**：支持查看所有历史变更记录
- **版本控制**：保留项目信息的历史版本

#### 2.1.3 界面设计

**项目列表页**
- 表格形式展示项目列表
- 支持按项目名称、状态、承接日期筛选
- 支持分页显示，每页20条记录
- 提供新建、编辑、查看、删除操作

**项目详情页**
- Tab页面设计：基本信息、财务信息、人员配置、文档管理
- 表单形式展示和编辑项目信息
- 显示项目状态变更历史
- 提供快速操作按钮

#### 2.1.4 业务规则

**数据校验**
- 项目名称不能重复
- 承接日期不能晚于当前日期
- 项目编号格式：PRJ-YYYYMMDD-XXX

**权限控制**
- 项目创建：项目经理以上权限
- 项目编辑：项目负责人或管理员
- 项目查看：所有用户
- 项目删除：仅管理员

### 2.2 经营信息管理

#### 2.2.1 功能概述
**目标**：管理客户信息和经营人员分配，维护客户关系，明确经营责任。

#### 2.2.2 核心功能

**客户信息管理**
- **甲方名称**：文本输入，必填项，最大长度100字符
- **联系人姓名**：文本输入，必填项，最大长度20字符
- **联系电话**：文本输入，格式校验，必填项
- **客户地址**：文本输入，选填项，最大长度200字符
- **客户类型**：下拉选择（政府机关、企事业单位、私人业主）

**经营人员管理**
- **经营负责人**：下拉选择，从用户列表中选择
- **经营人员**：多选下拉，支持选择多个经营人员
- **责任分工**：文本描述经营人员的具体分工
- **联系记录**：记录与客户的沟通历史

#### 2.2.3 界面设计

**客户列表页**
- 卡片式布局展示客户信息
- 支持按客户名称、类型搜索
- 显示客户基本信息和最近联系时间
- 提供新建、编辑、查看操作

**客户详情页**
- 客户基本信息展示和编辑
- 关联项目列表显示
- 联系记录时间轴展示
- 经营人员分配管理

#### 2.2.4 业务规则

**数据校验**
- 联系电话格式校验（手机号或固话）
- 甲方名称不能为空
- 经营负责人必须指定

**关联关系**
- 客户与项目一对多关系
- 经营人员与客户多对多关系
- 支持客户信息变更历史追踪

### 2.3 合同财务管理

#### 2.3.1 功能概述
**目标**：管理合同信息和财务收支，跟踪应收账款，监控财务状况。

#### 2.3.2 核心功能

**合同信息管理**
- **合同文本上传**：支持PDF、Word格式，最大50MB
- **补充协议上传**：支持多个补充协议文件
- **签订时间**：日期选择器，必填项
- **合同费率**：百分比输入，支持小数点后2位
- **合同金额**：分别录入设计费、勘察费、咨询费

**收支记录管理**
- **应收金额**：根据合同金额自动计算或手动录入
- **开票记录**：开票时间、开票金额、发票号码
- **收款记录**：支付时间、支付金额、支付方式
- **未收金额**：自动计算应收减去已收
- **管理费比例**：公司管理费扣除比例

#### 2.3.3 界面设计

**合同列表页**
- 表格展示合同基本信息
- 显示合同状态（待收款、部分收款、已完成）
- 支持按项目、签订时间筛选
- 提供合同金额统计汇总

**合同详情页**
- 合同基本信息展示
- 收支明细表格展示
- 文件上传和下载功能
- 应收账款提醒功能

#### 2.3.4 业务规则

**金额计算**
- 未收金额 = 应收金额 - 已收金额
- 管理费 = 应收金额 × 管理费比例
- 支持多次收款记录

**状态管理**
- 根据收款情况自动更新合同状态
- 超期应收自动标红提醒
- 支持收款计划制定

### 2.4 项目概算管理

#### 2.4.1 功能概述
**目标**：管理项目概算信息和批复审计资料，支持概算与实际对比分析。

#### 2.4.2 核心功能

**概算信息管理**
- **估算金额**：分别录入设计费、勘察费、咨询费概算
- **概算依据**：文本描述概算的计算依据
- **概算时间**：概算编制时间
- **概算人员**：概算编制人员

**批复审计管理**
- **批复报告上传**：支持PDF格式文件上传
- **审计报告上传**：支持PDF格式文件上传
- **批复金额**：批复的设计费、勘察费、咨询费
- **审计金额**：审计确认的设计费、勘察费、咨询费

#### 2.4.3 界面设计

**概算列表页**
- 按项目展示概算信息
- 显示概算金额和批复金额对比
- 支持按项目名称搜索
- 提供概算统计分析

**概算详情页**
- 概算信息录入表单
- 批复审计文件上传
- 概算与实际金额对比图表
- 差异分析说明

#### 2.4.4 业务规则

**数据关联**
- 概算信息与项目一对一关系
- 批复金额不能超过概算金额的120%
- 支持概算调整记录

### 2.5 招投标管理

#### 2.5.1 功能概述
**目标**：管理招投标过程文档和专家费用，规范招投标流程。

#### 2.5.2 核心功能

**投标文档管理**
- **招标文本上传**：招标文件PDF上传
- **投标文本上传**：投标文件PDF上传
- **中标文本上传**：中标通知书PDF上传
- **文档版本管理**：支持文档版本控制

**专家费用管理**
- **专家信息**：专家姓名、联系方式、专业领域
- **费用支付**：现金支付金额、转账支付金额
- **支付人员**：记录转账支付的具体人员
- **支付凭证**：支付凭证文件上传

#### 2.5.3 界面设计

**招投标列表页**
- 按项目展示招投标信息
- 显示投标状态（投标中、已中标、未中标）
- 支持按时间、状态筛选
- 提供专家费用统计

**招投标详情页**
- 文档上传和下载功能
- 专家费用明细管理
- 投标过程时间轴展示
- 相关费用统计

#### 2.5.4 业务规则

**文档管理**
- 文档上传大小限制50MB
- 支持PDF格式文档
- 文档下载权限控制

### 2.6 人员配置管理

#### 2.6.1 功能概述
**目标**：管理项目人员分工，明确各专业人员职责，规范协作流程。

#### 2.6.2 核心功能

**项目人员配置**
- **项目负责人**：下拉选择，必填项
- **专业配置**：8个专业领域的人员配置
  - 专业1-8：道路、桥梁、排水、交通、照明、绿化、景观、其他
  - 每个专业配置：设计人、参与人、复核人
- **审核审定人员**：审核人、审定人配置

**人员工作量统计**
- **项目参与度**：统计人员参与的项目数量
- **专业分布**：统计人员在各专业的分布情况
- **工作负荷**：评估人员当前工作负荷

#### 2.6.3 界面设计

**人员配置页**
- 表格形式展示8个专业的人员配置
- 下拉选择框选择人员
- 支持批量配置和复制配置
- 显示人员工作量预警

**人员统计页**
- 人员工作量图表展示
- 专业分布饼图展示
- 人员效率分析报表
- 工作量平衡建议

#### 2.6.4 业务规则

**人员配置规则**
- 每个专业必须配置设计人
- 复核人不能是设计人本人
- 项目负责人必须参与项目
- 支持人员变更记录

### 2.7 生产支出管理

#### 2.7.1 功能概述
**目标**：管理项目生产过程中的各项费用支出，准确核算项目成本。

#### 2.7.2 核心功能

**用车费用管理**
- **用车时间**：出车日期和时间
- **行驶公里数**：起始公里数、结束公里数
- **费用计算**：按每公里2元自动计算
- **用车人员**：记录用车人员信息
- **用车目的**：记录用车事由

**差旅费用管理**
- **住宿费用**：住宿时间、住宿地点、费用金额
- **交通费用**：交通方式、起止地点、费用金额
- **餐饮费用**：就餐时间、就餐地点、费用金额
- **凭证上传**：支持发票凭证图片上传

#### 2.7.3 界面设计

**费用录入页**
- Tab页面设计：用车费用、住宿费用、交通费用
- 表单录入，支持快速录入
- 费用自动计算和汇总
- 凭证照片上传预览

**费用统计页**
- 费用分类统计图表
- 按项目、人员、时间维度统计
- 费用趋势分析
- 成本控制建议

#### 2.7.4 业务规则

**费用计算**
- 用车费用 = 公里数 × 2元/公里
- 支持费用标准配置
- 费用审批工作流

### 2.8 文档存档管理

#### 2.8.1 功能概述
**目标**：统一管理项目相关文档，支持文档分类存储、检索和共享。

#### 2.8.2 核心功能

**文档分类管理**
- **合同文档**：合同、补充协议、招投标文件
- **设计文档**：初步设计、施工图设计、图纸审核单
- **财务文档**：发票、收据、付款凭证

**文档操作功能**
- **文档上传**：支持拖拽上传，批量上传
- **文档下载**：单个下载、批量下载
- **文档预览**：在线预览PDF、图片格式
- **文档搜索**：按文档名称、类型、上传时间搜索

#### 2.8.3 界面设计

**文档管理页**
- 左侧文档分类树形菜单
- 右侧文档列表网格显示
- 文档操作工具栏
- 文档搜索过滤功能

**文档详情页**
- 文档基本信息展示
- 文档预览功能
- 文档版本历史
- 下载统计信息

#### 2.8.4 业务规则

**文档权限**
- 上传权限：项目参与人员
- 查看权限：项目相关人员

## 五、接口设计规范

### 5.1 API设计规范

#### URL规范
```
GET    /api/projects              # 获取项目列表
GET    /api/projects/{id}         # 获取项目详情
POST   /api/projects              # 创建项目
PUT    /api/projects/{id}         # 更新项目
DELETE /api/projects/{id}         # 删除项目
```

#### 请求响应格式
```json
// 请求格式
{
  "projectName": "城市道路改造项目",
  "acceptDate": "2024-01-15",
  "projectDesc": "项目描述",
  "designUnit": "某某设计院",
  "projectCode": "PRJ-20240115-001"
}

// 响应格式
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "id": 1,
    "projectName": "城市道路改造项目",
    "status": "进行中",
    "createTime": "2024-01-15 10:00:00"
  }
}
```

### 5.2 错误处理规范

#### 错误码定义
- **200**：操作成功
- **400**：请求参数错误
- **401**：未授权访问
- **403**：权限不足
- **404**：资源不存在
- **500**：服务器内部错误

#### 错误响应格式
```json
{
  "code": 400,
  "message": "请求参数错误",
  "error": "项目名称不能为空"
}
```
