### 路由与认证规范

#### 路由格式

所有 API 请求遵循统一的路由格式：`/{service}/{version}/{auth_level}/{path}`

**路由组成**:
- **service**: 服务名称，本服务为 `timejourney`
- **version**: API 版本，本服务使用 `v1`
- **auth_level**: 认证级别，本服务支持 `public` 和 `user`
- **path**: 具体接口路径

**示例路由**:
- `GET /timejourney/v1/public/health` - 健康检查（无需认证）
- `GET /timejourney/v1/user/locations` - 查询地点列表（需要JWT认证）
- `POST /timejourney/v1/user/locations` - 创建地点（需要JWT认证）
- `WS /timejourney/v1/user/ws` - WebSocket连接（需要JWT认证）

#### 认证级别

本服务支持两种认证级别：

| 认证级别 | 说明 | 认证要求 | 适用场景 |
|---------|------|---------|---------|
| **public** | 公开接口 | 无需认证 | 健康检查 |
| **user** | 用户接口 | JWT Token 认证 | 所有业务接口、WebSocket连接 |

**认证流程**:
1. **public**: 网关直接转发请求，不进行任何认证
2. **user**: 网关验证 JWT Token，成功后注入用户信息到请求头

#### Header 转发机制

**网关转发 Header（认证成功后注入）**:

对于 `user` 级别认证，外部服务（HTTP）会收到以下 header：

```
X-User-ID: <user_id>                    # 用户ID（UUID格式）
X-User-Username: <username>              # 用户名
X-User-AppID: <app_id>                   # 应用ID
X-User-SessionID: <session_id>            # 会话ID（UUID格式）
```

**重要说明**:
1. **Header可用性**: 作为外部服务（HTTP），系统不会收到 `X-User-Role` 和 `X-User-IsAdmin` header（仅本地服务可接收）
2. **UUID格式**: `X-User-ID` 和 `X-User-SessionID` 都是 UUID v4 格式的字符串，必须作为字符串处理
3. **用户信息提取**: 系统必须从 Header 中读取用户信息，用于数据隔离和权限验证

### 部署规范

#### 本地开发部署脚本

本地开发部署脚本（`dev.sh`）提供完整的本地Docker环境部署流程：

**功能范围**:
1. **构建Docker镜像**: 根据 Dockerfile 构建应用镜像
2. **启动容器**: 使用 docker-compose 启动服务容器（应用、数据库等）
3. **环境配置**: 
   - 数据库初始化（创建数据库、执行初始化SQL）
   - 环境变量配置（从 .env 文件加载）
4. **健康检查**: 验证服务是否正常启动，检查关键接口是否可访问

**使用场景**: 开发人员在本地快速启动完整的开发环境

#### 生产部署脚本

生产部署脚本（`deploy.sh`）支持将项目部署到生产服务器的Docker环境：

**部署流程**:
1. **本地构建**: 在本地构建Docker镜像
2. **文件传输**: 通过SSH免密连接将构建产物传输到生产服务器
3. **远程部署**: 在服务器上执行部署命令（停止旧容器、启动新容器）
4. **数据库迁移**: 自动执行数据库迁移，应用数据库结构变更
5. **健康检查**: 验证新版本是否正常运行
6. **回滚支持**: 如果部署失败，自动回滚到上一个版本

**配置方式**:
- **环境变量**: 通过环境变量传递敏感信息（如SSH密钥路径、服务器地址等）
- **配置文件**: 通过 `.env` 或 `config.yaml` 文件管理通用配置（如数据库连接、服务端口等）

**SSH免密连接**:
- 使用SSH密钥认证，无需输入密码
- 支持通过环境变量或配置文件指定SSH密钥路径
- 支持指定SSH用户和服务器地址

**回滚机制**:
- 部署前自动备份当前版本
- 部署失败时自动回滚到上一个版本
- 支持手动触发回滚操作

**数据库迁移**:
- 部署时自动检测数据库结构变更
- 使用数据库迁移工具（如 golang-migrate）自动应用变更
- 支持迁移回滚（如果需要）
