## Issue Details

### Issues Found

#### [P1-001] GORM Distinct用法不正确 - 发放人数统计
- **File**: backend/internal/services/financial_service.go:398
- **Category**: Business Logic
- **Description**: 使用 `Distinct("recipient_id").Count()` 在 GORM 中可能不会正确工作。应该先使用 `Pluck` 获取去重后的值，然后计算长度。
- **Impact**: 发放人数统计可能不准确
- **Status**: [X] FIXED
- **Fix**: 改为使用 `Pluck("recipient_id", &distinctRecipients)` 获取去重后的 recipient_id 列表，然后使用 `len(distinctRecipients)` 计算数量
- **Commit**: 51edd20d

#### [P1-002] 统计信息查询未失效 - 数据变更后统计不更新
- **File**: frontend/src/components/shared/BonusAllocationForm.tsx, BonusAllocationList.tsx
- **Category**: State Synchronization
- **Description**: 当创建、更新或删除生产奖金记录时，没有失效 `productionBonusStatistics` 查询，导致统计信息不会自动刷新。
- **Impact**: 用户操作后统计信息不会立即更新，需要手动刷新页面
- **Status**: [X] FIXED
- **Fix**: 在 createMutation、updateMutation 和 deleteMutation 的 onSuccess 回调中添加 `queryClient.invalidateQueries({ queryKey: ['productionBonusStatistics', projectId] })`
- **Commit**: 51edd20d

#### [P0-001] TypeScript 类型错误 - ProjectMemberUserSummary 没有 email 属性
- **File**: frontend/src/components/shared/BonusAllocationForm.tsx:93
- **Category**: Data Structure Alignment
- **Description**: 代码尝试访问 `m.user?.email`，但 `ProjectMemberUserSummary` 类型定义中没有 `email` 属性，只有 `username` 和 `real_name`。
- **Impact**: 前端构建失败，TypeScript 编译错误
- **Status**: [X] FIXED
- **Fix**: 移除对 `email` 的引用，只使用 `username` 作为后备值
- **Commit**: 51edd20d
