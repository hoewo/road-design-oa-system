### ⚠️ 发现的问题

#### 问题1: 前端创建/更新甲方时未传递 project_id
**位置**: 
- `frontend/src/components/client/ClientForm.tsx`
- `frontend/src/services/project.ts`

**问题描述**:
在项目上下文中创建或更新甲方时，前端没有传递 `project_id` 查询参数，导致后端无法进行权限检查。

**影响**:
- 任何已登录用户都可以在项目上下文中创建/更新甲方，绕过了权限检查
- 违反了业务规则：只有经营负责人、项目管理员或系统管理员才能管理项目经营信息

**当前代码**:
```typescript
// frontend/src/services/project.ts
createClient: async (data: CreateClientRequest): Promise<Client> => {
  return post<Client>('/user/clients', data)  // ❌ 没有传递 project_id
}

// frontend/src/components/client/ClientForm.tsx
createMutation.mutate(values)  // ❌ 没有传递 project_id
```

**修复方案**:
1. 修改 `projectService.createClient` 和 `updateClient` 支持可选的 `projectId` 参数
2. 修改 `ClientForm` 组件接收 `projectId` prop
3. 在 `ClientSelectModal` 中调用 `ClientForm` 时传递 `projectId`
4. 修改 `post` 和 `put` API 函数支持查询参数

#### 问题2: ClientForm 中仍包含联系人字段
**位置**: `frontend/src/components/client/ClientForm.tsx`

**问题描述**:
`ClientForm` 组件中仍然包含 `contact_name` 和 `contact_phone` 字段，但根据需求，联系人信息应该通过 `ProjectContact` 实体管理，而不是在 `Client` 中。

**影响**:
- 数据模型不一致
- 可能造成用户困惑：联系人信息应该在哪里管理

**当前代码**:
```typescript
// frontend/src/components/client/ClientForm.tsx
<Form.Item name="contact_name" label="联系人">
  <Input placeholder="请输入联系人" />
</Form.Item>
<Form.Item name="contact_phone" label="联系电话">
  <Input placeholder="请输入联系电话" />
</Form.Item>
```

**修复方案**:
从 `ClientForm` 中移除 `contact_name` 和 `contact_phone` 字段，这些信息应该在 `ClientSelectModal` 中管理。

#### 问题3: API 函数不支持查询参数
**位置**: `frontend/src/services/api.ts`

**问题描述**:
`post` 和 `put` 函数不支持查询参数，无法在 POST/PUT 请求中传递 `project_id`。

**当前代码**:
```typescript
export const post = async <T>(url: string, data?: any): Promise<T> => {
  const response = await api.post<any>(url, data)  // ❌ 没有支持 params
  // ...
}
```

**修复方案**:
修改 `post` 和 `put` 函数支持可选的查询参数：
```typescript
export const post = async <T>(url: string, data?: any, params?: any): Promise<T> => {
  const response = await api.post<any>(url, data, { params })
  // ...
}
```

#### 问题4: 权限检查逻辑不一致
**位置**: `backend/internal/services/client_service.go`

**问题描述**:
当 `projectID` 为空时，`CreateClient`、`UpdateClient`、`DeleteClient` 方法不进行权限检查，注释说明"假设在 handler 层检查了权限"，但实际上 handler 层也没有强制要求。

**影响**:
- 如果前端不传递 `project_id`，权限检查会被绕过
- 全局创建/更新/删除甲方时没有权限控制

**修复方案**:
考虑两种方案：
1. **方案A（推荐）**: 在项目上下文中强制要求传递 `project_id`，全局操作需要管理员权限
2. **方案B**: 在 handler 层添加权限检查中间件
