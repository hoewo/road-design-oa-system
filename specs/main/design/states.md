## 全局状态

### 1. 加载中 (Loading)

**定义**: 应用正在加载数据或执行操作的状态。

**触发条件**:
- 页面初始加载
- 数据刷新（列表、详情、统计等）
- 表单提交
- 文件上传/下载
- API 请求进行中

**视觉表现**:
- **指示器**: Ant Design Spin 组件（旋转的圆形指示器）
- **位置**: 
  - 页面级：居中显示，覆盖整个页面
  - 组件级：组件内部显示（如表格 loading 属性）
  - 按钮级：按钮内显示（confirmLoading 属性）
- **遮罩**: 页面级加载显示半透明遮罩
- **提示文本**: 通常不显示文本，仅显示加载动画

**交互行为**:
- 加载期间禁用相关操作按钮
- 用户无法进行其他操作（页面级加载）
- 可以取消操作（如需要）

**退出条件**:
- 数据加载完成
- 操作执行完成
- 用户取消操作（如支持）
- 发生错误

**相关组件**:
- `Spin` - Ant Design 加载指示器
- `Table.loading` - 表格加载状态
- `Button.loading` - 按钮加载状态

**实现示例**:
```tsx
// 页面级加载
{loading && (
  <div style={{ display: 'flex', justifyContent: 'center', alignItems: 'center', minHeight: '100vh' }}>
    <Spin size="large" />
  </div>
)}

// 表格加载
<Table loading={isLoading} columns={columns} dataSource={data} />

// 按钮加载
<Button loading={isSubmitting} onClick={handleSubmit}>
  提交
</Button>
```

### 2. 错误状态 (Error)

**定义**: 应用遇到错误无法继续执行的状态。

**触发条件**:
- API 请求失败（网络错误、服务器错误等）
- 数据验证失败
- 权限不足
- 文件上传失败
- 操作执行失败

**视觉表现**:
- **消息提示**: Ant Design message.error
  - 位置：页面右上角
  - 样式：红色背景，白色文字
  - 显示时长：3秒自动消失
- **表单错误**: 输入框下方显示红色错误文字（12px）
- **错误页面**: 显示错误图标、标题、描述、操作按钮（如需要）

**交互行为**:
- 显示错误消息
- 表单错误阻止提交
- 提供重试操作（如适用）
- 用户可以关闭错误提示

**退出条件**:
- 用户点击重试并成功
- 用户修正错误（表单验证）
- 错误消息自动消失（3秒后）
- 用户手动关闭错误提示

**相关组件**:
- `message.error` - Ant Design 错误消息
- `Form.Item` - 表单验证错误显示
- `Alert` - 错误提示框（如需要）

**实现示例**:
```tsx
// 错误消息
message.error(error.message || '操作失败')

// 表单错误
<Form.Item
  name="field"
  rules={[{ required: true, message: '请填写此字段' }]}
>
  <Input />
</Form.Item>
```

### 3. 空状态 (Empty)

**定义**: 列表或数据为空的状态。

**触发条件**:
- 列表数据为空
- 搜索结果为空
- 文件列表为空
- 统计数据为空

**视觉表现**:
- **表格空状态**: Ant Design Table 默认空状态
  - 显示"暂无数据"文本
  - 居中显示
- **自定义空状态**: 
  - 图标（可选）
  - 标题文本
  - 描述文本
  - 操作按钮（引导用户添加数据）

**交互行为**:
- 显示空状态提示
- 提供添加数据入口（如有权限）
- 用户可以执行添加操作

**退出条件**:
- 用户添加数据后列表有内容
- 数据刷新后列表有内容

**相关组件**:
- `Table` - 表格空状态（Ant Design 默认）
- `Empty` - Ant Design 空状态组件（如需要）

**实现示例**:
```tsx
// 表格空状态（自动处理）
<Table columns={columns} dataSource={[]} />

// 自定义空状态
{data.length === 0 && (
  <div style={{ textAlign: 'center', padding: '60px 20px', color: '#999' }}>
    <div>暂无数据</div>
    {canManage && (
      <Button type="primary" onClick={handleAdd}>
        添加数据
      </Button>
    )}
  </div>
)}
```

## 界面特定状态

### 经营信息统计 状态

#### 状态1: 正常显示

**定义**: 统计数据正常显示的状态。

**触发条件**:
- 统计数据加载成功
- 日期筛选后数据加载成功

**视觉表现**:
- 显示三个统计项（总应收金额、已收金额、未收金额）
- 每个统计项显示标题和数值
- 数值使用语义颜色：
  - 总应收金额：蓝色 (#1890ff)
  - 已收金额：绿色 (#52c41a)
  - 未收金额：红色 (#ff4d4f)
- 显示统计时间范围说明（如有筛选）

**交互行为**:
- 可以修改日期筛选
- 可以重置筛选
- 统计数据实时更新

**退出条件**:
- 用户修改日期筛选
- 数据刷新

**状态转换**:
```
[加载中] → [数据加载成功] → [正常显示]
```

---

#### 状态2: 日期筛选

**定义**: 用户设置了日期筛选条件的状态。

**触发条件**:
- 用户选择开始日期或结束日期

**视觉表现**:
- 日期选择器显示选中的日期
- 显示统计时间范围说明：
  - 格式："统计时间范围：从 YYYY-MM-DD 到 YYYY-MM-DD"
  - 颜色：#666
  - 字体大小：12px

**交互行为**:
- 可以修改日期
- 可以重置筛选
- 统计数据根据筛选条件更新

**退出条件**:
- 用户重置筛选
- 用户修改日期

**状态转换**:
```
[正常显示] → [选择日期] → [日期筛选]
[日期筛选] → [重置] → [正常显示]
```

## 状态转换图

### 经营信息页面 状态转换

```
[页面加载]
    ↓
[加载中]
    ├→ [数据加载成功] → [正常显示]
    │      ├→ [用户交互] → [编辑状态]
    │      │      ↓
    │      │   [保存成功] → [正常显示]
    │      │
    │      └→ [日期筛选] → [筛选显示]
    │             ↓
    │          [重置] → [正常显示]
    │
    └→ [数据加载失败] → [错误状态]
           ↓
        [重试] → [加载中]
```

### 列表页面 状态转换

```
[页面加载]
    ↓
[加载中]
    ├→ [数据加载成功] → [有数据] → [正常显示]
    │      │                    ↓
    │      │                [用户操作] → [操作中]
    │      │                    ↓
    │      │                [操作成功] → [正常显示]
    │      │
    │      └→ [数据为空] → [空状态]
    │                          ↓
    │                      [添加数据] → [正常显示]
    │
    └→ [数据加载失败] → [错误状态]
           ↓
        [重试] → [加载中]
```

## 权限状态处理

### 有权限状态
- 显示所有操作按钮
- 可以执行所有操作
- 显示编辑入口

### 无权限状态
- **可查看但隐藏编辑入口**:
  - 完全隐藏所有编辑入口（添加、编辑、删除按钮/链接等）
  - 不显示禁用状态的按钮
  - 不显示"无权限操作"提示文本
  - 可以查看所有内容（列表、详情、文件等）
- **完全不可访问**:
  - 系统拒绝访问
  - 显示权限不足提示
