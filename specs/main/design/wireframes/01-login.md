# 登录页面 线框图

## 界面信息

- **界面名称**: 登录页面
- **View 名称**: `LoginView`
- **平台**: Web（跨平台）
- **导航层级**: 第1层（入口页面）
- **前置界面**: 无（系统入口）
- **后续界面**: 项目列表页面、用户管理页面（根据权限）

## 用户流程

### 流程描述

用户通过验证码登录系统，支持邮箱和手机号两种方式：

1. 用户选择登录方式（邮箱/手机号）
2. 输入邮箱或手机号
3. 点击"获取验证码"按钮
4. 系统发送验证码到用户邮箱或手机
5. 用户输入收到的验证码
6. 点击"登录"按钮
7. 系统验证验证码，登录成功跳转到主页面

### 流程图

```
无 → 登录页面（选择登录方式）
登录页面 → 项目列表页面（登录成功）
登录页面 → 登录页面（登录失败，显示错误）
```

## 数据需求

### 输入数据

- **登录方式**: `string` - 用户选择（"email" | "phone"）
- **邮箱/手机号**: `string` - 用户输入
- **验证码**: `string` - 用户输入（6位数字）

### 输出数据

- **Token信息**: `object` - 登录成功后返回
  - `access_token`: `string` - 访问令牌，存储到localStorage
  - `refresh_token`: `string` - 刷新令牌，存储到localStorage
- **用户信息**: `object` - 登录成功后返回
  - `user_id`: `string` - 用户ID
  - `username`: `string` - 用户名
  - `email`: `string` - 邮箱

### API调用

1. **发送验证码**:
   - 接口: `POST /auth-server/v1/public/send_verification`
   - 参数: `{code_type: "email"|"sms", target: "邮箱或手机号", purpose: "login"}`
   - 响应: `{success: true}`

2. **用户登录**:
   - 接口: `POST /auth-server/v1/public/login`
   - 参数: `{email/phone: "邮箱或手机号", code: "验证码", code_type: "email"|"sms", purpose: "login"}`
   - 响应: `{success: true, data: {tokens: {access_token, refresh_token}, user: {...}}}`

## 性能要求

- **界面渲染**: 首次加载时间 < 500ms
- **验证码发送**: 响应时间 < 2s
- **登录验证**: 响应时间 < 2s
- **交互响应**: 按钮点击响应时间 < 100ms

## 无障碍支持

- **键盘导航**: 支持Tab键切换输入框和按钮，Enter键提交
- **屏幕阅读器**: 所有输入框和按钮都有清晰的标签和提示
- **错误提示**: 错误信息通过ARIA live region实时播报
- **焦点管理**: 验证码输入框自动聚焦，错误时焦点回到错误输入框

## 实现注意事项

### 基于NebulaAuth的登录实现

1. **API配置**:
   - 开发环境: `NEBULA_AUTH_URL = 'http://your-aliyun-ip:8080'`
   - 生产环境: `NEBULA_AUTH_URL = 'http://your-aliyun-ip:8080'`（网关地址）
   - 登录接口直接调用NebulaAuth网关，不通过业务服务

2. **Token管理**:
   - 登录成功后，将`access_token`和`refresh_token`存储到localStorage
   - 所有业务接口请求自动携带`Authorization: Bearer <token>` Header
   - Token过期时自动刷新（通过请求拦截器处理）

3. **验证码管理**:
   - 验证码发送后，显示60秒倒计时
   - 倒计时期间禁用"重新发送"按钮
   - 倒计时结束后，允许重新发送验证码

4. **错误处理**:
   - 验证码发送失败: 提示用户检查邮箱/手机号
   - 验证码错误: 提示验证码错误或已过期，允许重新输入或重新发送
   - 账号不存在: 提示联系管理员创建账号
   - 网络错误: 提示检查网络连接，允许重试

5. **用户体验优化**:
   - 验证码输入框自动聚焦
   - 验证码输入完成后自动启用登录按钮
   - 登录成功后显示成功提示，然后自动跳转
   - 所有错误状态都提供明确的错误信息和解决方案

6. **安全性考虑**:
   - Token存储在localStorage，注意XSS攻击防护
   - 验证码有效期限制（由NebulaAuth控制）
   - 验证码输入错误次数限制（由NebulaAuth控制）

### 与现有系统的区别

- **旧系统**: 用户名+密码登录
- **新系统**: 邮箱/手机号+验证码登录（基于NebulaAuth）
- **不再支持**: 用户名密码登录、注册功能（系统不提供公开注册）
