# maintain-create-command

## 描述

专门用于优化和维护 `create-command.md` 命令的工具命令。

**核心功能**：根据用户的反馈和最佳实践，持续改进 `create-command.md` 的内容和结构。

**重要说明**：
- 优化目标唯一且固定为 `create-command.md`
- 用户输入中的任何改进建议都会应用到 `create-command.md` 上
- 即使用户的表述中提到本命令的名称，也是指使用本命令来优化 `create-command.md`
- 所有优化方案都需要人工确认后才会执行

## 参数

命令接受自然语言描述的需求，AI 会自动识别以下信息：

- **优化目标**：固定为 `create-command.md`（不可更改）
  - 用户输入中提到的任何步骤、内容、问题都默认指向 `create-command.md` 中的对应部分
  - 即使用户表述中包含其他命令文件名，也要理解为是使用本命令来优化 `create-command.md`
- **优化需求**：用户描述的具体优化建议或问题点
  - 可以是对某个步骤的改进建议
  - 可以是对整体结构的调整意见
  - 可以是对用户体验的优化想法
- **优化背景**：用户补充的背景信息或使用场景（可选）
- **优化深度**：优化范围（可选：全面优化/局部调整）

## 执行步骤

1. **确认优化目标并解析用户输入**
   - **核心原则**：此命令的优化目标固定且唯一是 `.cursor/commands/create-command.md`，永远不会优化其他命令
   - **输入解析规则**：
     * 用户提到的任何"步骤"、"内容"、"问题"都是指 `create-command.md` 中的相应部分
     * 即使用户的表述中提到 `maintain-create-command.md`，也应理解为这是调用本命令来优化 `create-command.md`
     * 用户的需求描述应被理解为对 `create-command.md` 的优化建议
   - **明确向用户确认**：
     * 首先明确告知用户："我将优化 `create-command.md` 命令"
     * 然后复述用户的优化需求，例如："您希望优化 create-command.md 中的第X步..."
     * 如果用户明确纠正说要优化其他命令，终止流程并说明此命令仅用于维护 `create-command.md`

2. **搜索最佳实践**
   - 从 Cursor 官方文档搜索最新的 Command 规范
   - 从第三方资源查找 Command 编写最佳实践
   - 汇总关键要点和改进建议

3. **读取 create-command**
   - 读取 `.cursor/commands/create-command.md` 文件内容
   - 分析当前命令的结构、完整性和清晰度
   - 识别与最佳实践的差距

4. **分析优化点**
   - 对比最佳实践，列出具体优化建议
   - 结合用户补充的背景信息
   - 评估优化的必要性和优先级
   - 重点关注：规范定义、执行步骤完整性

5. **提出澄清问题**（如需要）
   - 如果优化方向不明确，提出具体问题
   - 询问用户对命令规范的偏好
   - 确认优化范围和深度

6. **生成优化方案**
   - 展示当前版本与优化版本的对比
   - 说明每个改动的理由
   - 提供清晰的变更摘要
  
7. **等待人工确认**
   - 明确说明"需要人工确认是否执行优化"
   - 展示完整的优化内容预览
   - 等待用户批准或调整方案

8. **执行优化**（获得确认后）
   - 更新 `create-command.md` 文件
   - 验证文件格式正确性
   - 确认优化已生效

## 注意事项

- **⚠️ 关键：此命令仅维护 `create-command.md`**
  - 无论用户如何表述，优化目标永远是 `create-command.md`
  - 用户提到的步骤、内容、问题都指向 `create-command.md`
  - 即使输入中包含 `maintain-create-command.md`，也是指使用本命令优化 `create-command.md`
  - 绝不会优化 `maintain-create-command.md` 本身或其他命令文件
- **在执行步骤1必须明确向用户确认优化目标是 `create-command.md`**
- 必须先搜索最新的官方最佳实践，不依赖历史知识
- 所有优化必须经过人工确认，不可自动执行
- 优化建议要具体明确，避免模糊表述
- 保持 `create-command` 的核心功能和向后兼容性
- 如果没有必要优化，应明确说明并终止流程
