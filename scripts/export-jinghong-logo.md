# 京宏 Logo：从 PDF 到登录页 PNG 的导出说明

本文说明 `export-jinghong-logo.sh` 的设计思路、与人工/AI 协作中的分工，以及脚本对其他 PDF 的复用范围。

---

## 1. 背景：为什么不用「直接栅格化 PDF」

本项目的 `京宏logo.pdf` 存在典型问题：

- **MediaBox** 为整页（如 `0 0 2048 2048`），但 **CropBox** 更小，只框住画面的一部分。
- macOS `sips`（以及多数按 CropBox 渲染的工具）会**按 CropBox 裁切**，导致左侧、底部笔画被切掉。
- 页内实际是一张 **内嵌 JPEG**（`/Filter/DCTDecode`），变换矩阵铺满 MediaBox，**完整图形在这张位图里**。

因此最终方案是：**从 PDF 二进制中抽出第一张 DCT 图像流**，在像素域处理，而不是对整页 PDF 做栅格化。

---

## 2. 脚本核心思路（从原始 PDF 到最终 PNG）

整体是一条**确定性流水线**，输入固定路径的 PDF，输出带透明底的 PNG。

| 步骤 | 做什么 |
|------|--------|
| **① 提取 JPEG** | 在 PDF 字节中定位 `/Filter/DCTDecode` 后的 `stream … endstream`，取出合法 JPEG（`FF D8` 开头），用 Pillow 解码为 RGB。 |
| **② 估计背景色** | 用全图四角小块像素的 RGB 均值作为「浅灰底」参考，供后续抠透明使用。 |
| **③ 缩略预览 + 笔画掩膜** | 将全图缩到宽约 480px，在预览上构造前景掩膜：**红色笔画**（R 明显大于 G/B 且 R 足够高）与 **深灰结构**（桥墩等，RGB 均较低且 R 不太低）。二者并集为「主体骨架」。 |
| **④ 形态学膨胀** | 对掩膜做 `MaxFilter`（近似方形膨胀），把笔画之间的**白路、浅色块**包进包围盒，避免裁切时切掉内部细节。核大小随预览宽度在约 21～41 之间取奇数。 |
| **⑤ 映射回全分辨率裁切** | 将预览上的包围盒按 `scale_x/scale_y` 映射回原图坐标，并加约 **0.7% 边长（至少 10px）** 的安全边，从原 JPEG 裁出 ROI。 |
| **⑥ 浅灰底 → Alpha** | 在 ROI 内对每个像素算与背景色的 **max 通道差** `md`，在区间 `[t0, t1]`（当前约 13～24）上线性映射到 alpha，实现**半透明过渡**，减轻锯齿。 |
| **⑦ 二次收紧画布** | 对 alpha 做**行/列投影**：保留存在 `alpha > thr`（当前 36）的列与行，再外扩 2px，裁掉外圈弱透明边，使资源尺寸贴近「中间内容区」。 |
| **⑧ 输出尺寸** | 将 RGBA 图**等比缩放到目标宽度**（默认 480px，可由脚本第一个参数指定），写入 `frontend/public/branding/jinghong-logo.png`。 |

登录页 `Login.tsx` 中的 `img` **width/height** 需与导出后的**固有像素尺寸**一致，用于减少布局偏移（CLS）。

---

## 3. AI 与脚本分别做了什么

| 角色 | 内容 |
|------|------|
| **脚本（可重复执行）** | 实现上述整条流水线；路径、阈值、膨胀核、抠底区间、二次裁剪参数均写在仓库里，换 PDF 重跑即可再生图（在前提满足时）。 |
| **AI / 人工（交互与设计）** | 排查「CropBox 裁切」「内嵌 JPEG 与矢量混排」等现象；对比 `sips` 与抽 JPEG 的差异；迭代抠底与裁剪策略（例如从「全图 diff 掩膜」改为「红/灰笔画 + 预览图膨胀」以避免 JPEG 噪点撑满包围盒）；把结论固化进脚本与 `branding/README.md`；按需调整 `Login.tsx` 尺寸与样式。 |

**结论**：**像素级处理与文件生成由脚本负责**；**问题归因、参数调优与产品侧取舍由人/AI 在对话中完成**，再沉淀为代码与文档。

## 4. 能否直接复用到其他 PDF？

**不能指望「零改参数」通用于任意 PDF**。可作为**模板脚本**，在满足结构相似时少改即用；否则需改逻辑或换工具链。

### 4.1 脚本强依赖的前提（不满足则会失败或需大改）

1. **第一页、主视觉是一张内嵌 JPEG**  
   通过查找**第一个** `/Filter/DCTDecode` 流实现。若 logo 是矢量绘制、PNG/JBIG2、多图拼版、或 JPEG 不是主内容，提取会错或需重写解析逻辑。

2. **笔画颜色与当前启发式一致**  
   包围盒依赖 **红主导笔画 + 深灰辅色**。若品牌是蓝/绿/黑字、或灰底深字，必须改 `stroke` / `bridge` 的阈值或换成别的颜色规则。

3. **背景与四角采样一致**  
   背景色来自**四角**均值。若角上是装饰、渐变、出血图而非留白，背景估计会偏，`md` 与 alpha 会错，需改采样区域或改用固定色键。

4. **单页、单主图**  
   未处理多页 PDF、未在多个 DCT 流中选择「哪一张是 logo」。

### 4.2 可能需要微调的部分（同一类稿件换文件）

- 红/灰阈值（`130`、`115`、`28` 等）、膨胀核范围、`t0/t1`、`thr`、安全边 `pad_f`。
- 输出路径与 `Login.tsx` 中的宽高。

### 4.3 若要通用化，大致方向

- 用 **PyMuPDF / pypdf** 等按页枚举 XObject，按尺寸或名称选图。  
- 或统一要求设计方导出 **单页、单图、RGB、靠边留白** 的规范 PDF/SVG/PNG，减少启发式。

## 5. 使用方式（备忘）

```bash
# 仓库根目录；默认输出宽度 480
./scripts/export-jinghong-logo.sh

# 指定输出宽度（像素）
./scripts/export-jinghong-logo.sh 520
```

修改输入 PDF 时，编辑脚本内 `PDF=` 变量，或后续将路径改为参数化。
